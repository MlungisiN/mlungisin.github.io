Covid-19 Data Analysis

  • View Github

SQL Server Project

Selecting the Data that we will use

I utilized the SELECT statement and carefully selected the columns of data that I would need to use from the Covid-19 Deaths table to better understand the data that I would be querying whilst also simplifying the amount of columns that would be visible during my querying process.

Total Covid Cases vs Total Covid Deaths

I utilized the convert function to convert the number of deaths into a float value so I could then divide the total Covid 19 positive cases by Covid deaths multiplied by 100 to get the percentage of deaths. I used the Where clause to specify the location to South Africa to better gauge how badly the Covid pandemic affected our Country.

Total Covid Cases vs Population

I utilized the SELECT statement and specified the columns I would want to specifically use for this query whilst also running a division of total cases by population to view what percentage of the population was infected by the Covid virus within South Africa.

Countries with highest infection rate compared to Population

I made use of the aggregate function MAX to get the highest number of infections within different Countries and and then made use of division to divide the highest total cases by the countries population to give us a comparative analysis of Covid infections to population of the country as a percentage.

Total deaths in Continents

Made use of aggregate function to find the highest number of total number of deaths within the different Continents within the world, I utilized the ïs not null"in the where clause to ensure that deaths that weren't grouped into a specific continent would not show up as part of the query. I then grouped by continent and ordered it in descending order to ensure that the Continent with the highest death rate showed up 1st.

Countries with Highest Death Count per population

Made use of aggregate function to find the countries with the highest number of total deaths. I made use of the convert feature to convert number of deaths into an integer, I utilized the "is not null" in the where clause to ensure that deaths that weren't grouped into a specific continent would not show up as part of the query. I then grouped by location and ordered it in descending order to ensure that the country with the highest death rate would show up right at the top.

Global death percentage

Made use of the SUM aggregate function to calculate the total number of new cases and total number of new deaths and gave them an alias of TotalCases and TotalDeaths and then divided new deaths by the number of new covid cases and this gave us our global death percentage.

Total Population vs Vaccinations

I made the use of an inner join to join the two tables together so I could calculate the new vaccinations over total population, I joined using the location and date as these were the two items that could be identified and linked on both the two tables, I utilized the bigint feature as the number was to big to be recognized by SQL as a regular integer and kept giving an error message when executing the query.

Temp Table to calculate Total Population vs Vaccinations

I created a temp table that would calculate the total population vs vaccinations and used the DROP TABLE if exists feature to I could rerun the temp table without running into any issues whilst testing the queries, I then used a join within the temp table to join the Covid Deaths table to the Covid Vaccinations table to I could calsulat the percentage of Vaccinations to Total Population.

Nashville Housing Data Cleaning

  • VIEW GITHUB

SQL Server Project

Standardize the Date Time format

I made use of the Alter Table feature to add a column called SaleDateConverted and them populated that column using the initial date column that was present on the table but I converted it to use a more standardized date format as the initial one felt out of place and was less uniformed.

Populate Property Address Data

I made use of a self join to join the table to itself and the reason for this was to utilized data within the table to fill another column where some data was not filled in or listed as null, after completion of this join I then updated the table so that the new values could be visible on the main table moving forward so that I wouldn't have to rerun the same query on multiple occasions.

Separate the Address into separate columns (Address, City, State)

I selected the property address column then split it using the comma character as the point if splitting and I redid this using different string functions such as -1 to say it should split the string back to front and +1 to split it in the opposite direction and once this was done I modified the table to add property address column, property city and property state column and then populated these columns with the string characters that I had just split from the columns respectfully. The reason for this split was to remove the clutter of having everything in a singular column and it also allowed for one to be able to track the data by city, state and property alot easier.

Change Y and N to Yes and No in "Sold as Vacant" field

I selected the SoldAsVacant column where some of the values were listed as Y and N instead of Yes and No. I utilized the CASE statement to change Y to Yes and N to No and if it had any other value besides these two it should be listed as SoldAsVacant and upon this query working successfully I then modified it onto the table using the UPDATE statement and ran the query using SET so that the values would be changed permanently onto the table we are using.

Remove Duplicates

I created a CTE and selected all columns that I required for this query, I then used partition by so I could return a single value for each row, I ordered them using the Unique ID column so I could locate the duplicated info in the rows, I then selected everything from the CTE and used the drop column to delete the duplicated columns which were owner address, taxi district, property address and saledate columns.
